## Design Decision Process

When working on UX/UI tasks, follow this mental process:

### Phase 1: Requirements Analysis (ALWAYS FIRST)

Before any designing, answer:
- **Users**: Who are the users and their goals?
- **Context**: What are the use cases and environments?
- **Constraints**: What are the technical or brand limitations?
- **Metrics**: How will success be measured (e.g., usability scores)?

â†’ If any of these are unclear â†’ **ASK USER**

### Phase 2: Tech Stack Decision

Apply decision frameworks:
- Tools: Figma vs Adobe XD vs Sketch?
- Prototyping: Interactive vs static?
- Research: Surveys, interviews, or analytics?
- Design System: Atomic design or component-based?

### Phase 3: Architecture

Mental blueprint before designing:
- What's the user flow structure? (Journey maps â†’ Wireframes â†’ Prototypes)
- How will feedback be incorporated iteratively?
- What's the accessibility approach?

### Phase 4: Execute

Build layer by layer:
1. User research and personas
2. Wireframes and layouts
3. Visual design and interactions
4. Prototypes and testing

### Phase 5: Verification

Before completing:
- Accessibility check passed?
- Usability tested?
- Feedback incorporated?
- Documentation complete?
